PAGES & ROUTES

PROTECTED ADMIN ROUTES (role: admin):
GET  /dashboard/admin           → System overview
GET  /dashboard/admin/users     → User management
GET  /dashboard/admin/translations → Monitor translations

AUTHENTICATION:
POST /api/auth/[...nextauth]   (Supabase)

================================================================================
API INTEGRATION
================================================================================

BACKEND ENDPOINTS SUPPORTED:

User Endpoints:
  POST   /translations              → Upload PDF
  GET    /translations              → List translations
  GET    /translations/{id}         → Get translation details
  GET    /translations/{id}/download-pdf
  GET    /translations/{id}/download-epub
  GET    /user/stats                → User statistics

Admin Endpoints:
  GET    /admin/stats               → System statistics
  GET    /admin/users               → List all users
  POST   /admin/users               → Create user
  PATCH  /admin/users/{id}          → Update user (block/unblock)
  DELETE /admin/users/{id}          → Delete user
  GET    /admin/translations        → Get all translations

All endpoints automatically include JWT in Authorization header.

AUTHENTICATION FLOW
================================================================================

1. User navigates to /login
2. Enters email and password
3. Supabase authenticates credentials
4. JWT token generated and stored in secure HTTP-only cookie
5. User redirected to /dashboard
6. Middleware protects all routes except /login
7. All API calls automatically include JWT in Authorization header
8. Session persists across page reloads
9. User can logout, clearing session
10. Blocked users can't log in
